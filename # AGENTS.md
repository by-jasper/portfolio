# AGENTS.md

## Project Overview

This repository is for building a personal portfolio website for an HR professional based in Singapore.

The website should showcase:
- HR Operations and Payroll experience
- HR Shared Services knowledge
- HR process improvement work
- HR automation projects
- Responsible use of AI in HR
- A knowledge portal for HR professionals

The website is not just an online resume. It should position the owner as a forward-looking HR professional who understands HR operations, payroll, compliance, systems, process improvement, and AI-enabled automation.

## Main Goal

Build a clean, professional, recruiter-friendly personal portfolio website.

The website should help recruiters quickly understand:
1. Who I am
2. What HR experience I have
3. What problems I can solve
4. How I use automation and AI responsibly
5. How to contact me for job opportunities

## Preferred Style

Use a modern, clean, corporate design.

Design direction:
- Professional
- Trustworthy
- Modern
- Minimal
- Easy to read
- Slightly premium
- HR / technology / operations feel

Avoid:
- Overly colourful design
- Too many animations
- Gaming-style visuals
- Fancy effects that reduce readability
- Overly casual language
- Fake achievements
- Confidential company data
- Personal employee data

## Colour Direction

Preferred colour palette:
- Navy blue
- White
- Light grey
- Soft teal or soft gold as accent

The website should feel suitable for HR, corporate, technology, and transformation-related roles.

## Fonts

Use clean modern fonts such as:
- Inter
- Lato
- Source Sans Pro
- system-ui fallback

## Tone of Writing

The copywriting should be:
- Professional
- Clear
- Human
- Concise
- Recruiter-friendly
- Confident but not arrogant

Avoid:
- Too much jargon
- Long paragraphs
- Generic phrases such as “hardworking and responsible”
- Overclaiming
- Mentioning confidential company details

## Privacy and Data Protection Rules

This is very important.

Do not include:
- NRIC / FIN / passport numbers
- Employee names
- Salary data of identifiable employees
- Medical details
- Disciplinary matters
- Internal company documents
- Confidential company screenshots
- Internal approval emails
- Vendor-sensitive details
- Any restricted HR data

If sample data is required, use dummy data only.

## Build Quality

The site should be:
- Fully responsive
- Mobile-friendly
- Fast-loading
- Easy to navigate
- SEO-friendly
- Accessible where possible
- Easy to update later

## Technical Expectations

Use the existing project structure if the repo already has one.

If the repo is empty, build using a simple modern stack. Preferred:
- React / Next.js if already available
- Otherwise Vite + React is acceptable
- Use clean reusable components
- Use markdown or JSON content files where practical so the content can be updated easily

## Important Pages

Create these pages or sections:

1. Home
2. About
3. Experience
4. Case Studies
5. AI & HR Innovation
6. HR Knowledge Portal
7. Resume
8. Contact

A one-page layout is acceptable if it is clean and easy to navigate.

## Final Output Expected

When done:
- Make sure the site runs locally
- Fix lint/build errors
- Ensure navigation works
- Ensure all placeholder content is clearly marked
- Add comments where the owner should update personal details
- Do not expose secrets or private information
